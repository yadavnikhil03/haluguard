## Abstract

**HaluGuard** is a static analysis tool designed to identify common failure modes in code generated by Large Language Models (LLMs). As the adoption of AI coding assistants accelerates, developers frequently encounter "hallucinated" APIs—functions or classes synthesized by the model that do not exist in the referenced libraries—as well as insecure artifacts such as hardcoded credentials and incomplete implementation stubs. HaluGuard operates entirely locally, utilizing a deterministic rule-based engine and local knowledge bases to detect these anomalies prior to integration into production codebases.

## Methodology

HaluGuard parses unified diffs or individual source files across multiple programming languages, applying a suite of specialized detectors:

1. **Hallucinated API Detection**: Verifies imported modules and invoked functions against a known, localized database of valid standard library APIs for languages including TypeScript, Rust, Python, Java, C#, and PHP.
2. **Secret Identification**: Detects accidentally generated API keys, tokens, and credentials using high-confidence entropy analysis and regular expressions.
3. **Stub Detection**: Identifies placeholder logic (e.g., `TODO: implement`, `pass`, `throw new NotImplementedException()`) frequently emitted by LLMs to fulfill prompt requirements without providing functional implementation.

All analysis is performed client-side, ensuring complete data privacy with zero external telemetry or cloud dependencies.

## Installation and Usage

### Zero-install — run directly with npx

```sh
# Scan any file in your project instantly
npx haluguard src/index.ts

# Scan your entire AI-generated diff before merging
git diff origin/main | npx haluguard --stdin

# Try the demo — see what real detections look like
npx haluguard --help
```

No cloning. No config. No folders created in your project.

### Permanent global install

```sh
npm install -g haluguard

haluguard src/index.ts
haluguard src/auth.ts src/api.ts src/utils.ts
```

### Scan a git diff (recommended for AI-assisted projects)

This is the most powerful usage — scan only the lines your AI assistant actually wrote:

```sh
# Scan everything since your last commit
git diff HEAD~1 | haluguard --stdin

# Scan staged changes before committing
git diff --cached | haluguard --stdin

# In a PR — scan only lines changed vs main
git diff origin/main...HEAD | haluguard --stdin --fail-on high
```

### Pre-commit hook (auto-scan before every commit)

```sh
haluguard --init-hook
```

Installs a git hook that automatically scans your staged diff on every `git commit`. No setup required.

### GitHub Actions Integration

```yaml
name: HaluGuard Analysis
on: [pull_request]

jobs:
  analyze:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Run HaluGuard on PR diff
        run: |
          npx haluguard --version
          git diff origin/${{ github.base_ref }}...HEAD | npx haluguard --stdin --fail-on high
```

## Configuration and Suppression

False positives can be suppressed using inline directives:

```typescript
const myCustomMethod = library.customMethod(); // haluguard: ignore
```

A `.haluguard.yml` config file is also supported at the project root:

```yaml
min_severity: medium
fail_on: high
ignore:
  - "**/*.test.ts"
  - "vendor/**"
```

## VS Code Extension

HaluGuard ships with a VS Code extension that provides real-time diagnostics as you code. Hallucinated APIs, leaked secrets, and stubs appear as squiggly underlines directly in the editor.

```sh
npm run build
cd vscode-extension
npm install
npm run compile
```

Then press **F5** in VS Code to launch the Extension Development Host. See the [extension README](vscode-extension/README.md) for configuration options.
